Unvollendet – Lógica de Estados y Control de Flujo

Aplicación interactiva construida bajo un sistema estructurado paso a paso que combina manejo de lógica condicional avanzada, gestión de estados y toma de decisiones en base a variables.


Lo más destacado:

Diseñé una lógica de comportamiento para un oponente que analiza variables simples en tiempo real (como salud o recursos disponibles). No responde de forma aleatoria, sino que evalúa el contexto mediante árboles condicionales y probabilidades controladas para reaccionar de manera lógica.


También desarrollé un sistema de control por estados para gestionar los turnos y procesos lógicos paso a paso. Utilicé funciones asíncronas simples (coroutines) para bloquear las entradas del usuario cuando hay procesos activos, evitando así solapamientos de eventos e inconsistencias en el flujo.


Este proyecto refleja mi capacidad para resolver problemas lógicos, estructurar el flujo de una aplicación de forma segura y escribir un código ordenado y escalable.

Sistema de Arquitectura Desacoplada y Herramientas Auxiliares

Demo técnica centrada en la separación de responsabilidades en el código (Clean Code) y en la creación de herramientas que automatizan tareas dentro del entorno de desarrollo.


Separé estrictamente la lógica de la captura de datos (input), el procesamiento físico y las respuestas del sistema. Apliqué patrones de diseño comunes (como Singleton y Componentes) para crear módulos independientes que se pueden reutilizar en otros proyectos sin arrastrar dependencias.


El sistema maneja con precisión casos límite físicos e interpolaciones lógicas de tiempo, asegurando un rendimiento fluido y una respuesta rápida ante cualquier excepción o comportamiento inesperado del entorno.


Además, programé herramientas personalizadas integradas en el propio editor de desarrollo. Esto permite ajustar parámetros lógicos de forma visual sin necesidad de tocar o alterar líneas de código constantemente, mejorando el flujo de trabajo.


Un proyecto ideal para demostrar mi interés por las buenas prácticas, la organización limpia de archivos y la optimización básica del código.
